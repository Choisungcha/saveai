# 개인정보 영향평가 (PIA) 보고서

## SaveAI 금융 데이터 통합 서비스

### 1. 프로젝트 개요
- **프로젝트명**: SaveAI 금융 데이터 통합 서비스
- **목적**: 사용자의 금융 데이터를 안전하게 통합하고 분석하여 맞춤형 금융 상품 추천 및 지출 분석 서비스 제공
- **평가 대상**: 개인정보 처리 시스템 및 관련 프로세스
- **평가 기간**: 2026년 4월 23일 ~ 2026년 4월 30일

### 2. 개인정보 처리 활동
#### 2.1 수집하는 개인정보 항목
- **금융정보**:
  - 계좌번호 (마스킹 처리)
  - 신용카드 정보 (마스킹 처리)
  - 거래내역
  - 잔액 정보
- **개인식별정보**:
  - 이름 (마스킹 처리)
  - 이메일 주소 (마스킹 처리)
  - 연락처 정보

#### 2.2 개인정보 수집 방법
- 금융기관 API 연동 (토스페이먼츠, 카카오페이, KG이니시스)
- 사용자 직접 입력
- 서비스 이용 로그

#### 2.3 개인정보 처리 목적
- 금융 데이터 통합 및 분석
- 맞춤형 금융 상품 추천
- 지출 분석 및 절약 제안
- 서비스 개선 및 통계 분석

#### 2.4 개인정보 보유 및 이용 기간
- 서비스 이용 기간 동안 및 법령에 따른 보존 기간 (최대 5년)
- 회원 탈퇴 시 즉시 삭제 또는 법적 보존 기간 준수

### 3. 개인정보 처리 단계별 위험 평가

#### 3.1 수집 단계
- **위험**: 민감한 금융 데이터 수집으로 인한 정보 유출 가능성
- **완화 조치**:
  - API 통신 시 HTTPS/TLS 1.3 사용
  - 데이터 수집 즉시 암호화 적용
  - 최소 수집 원칙 준수

#### 3.2 저장 단계
- **위험**: 데이터베이스 해킹 또는 내부자 유출
- **완화 조치**:
  - AES-256 암호화 적용
  - 클라우드 HSM 사용
  - 접근 권한 최소화
  - 데이터베이스 암호화

#### 3.3 이용/처리 단계
- **위험**: 데이터 분석 과정에서의 정보 유출
- **완화 조치**:
  - 데이터 마스킹 적용
  - 감사 로그 기록
  - 처리 권한 검증

#### 3.4 제공 단계
- **위험**: 제3자 제공 시 정보 유출
- **완화 조치**:
  - 위탁 처리 계약 체결
  - 데이터 마스킹 적용
  - 제공 로그 기록

#### 3.5 파기 단계
- **위험**: 완전한 데이터 삭제 실패
- **완화 조치**:
  - 안전한 데이터 삭제 방법 사용
  - 삭제 로그 기록
  - 물리적/논리적 삭제 검증

### 4. 보안 조치 현황

#### 4.1 기술적 보안 조치
- **암호화**: AES-256, RSA, 클라우드 HSM
- **접근 제어**: 역할 기반 접근 제어 (RBAC)
- **네트워크 보안**: 방화벽, IDS/IPS, DDoS 방어
- **시스템 보안**: 정기 보안 패치, 취약점 스캔

#### 4.2 관리적 보안 조치
- **개인정보 보호 교육**: 연 2회 실시
- **접근 권한 관리**: 최소 권한 원칙 적용
- **감사 및 점검**: 연 2회 내부 감사
- **비상 대응 계획**: 개인정보 유출 대응 매뉴얼

#### 4.3 물리적 보안 조치
- **데이터센터**: 클라우드 서비스 (AWS/Azure/GCP) 사용
- **접근 통제**: 24시간 보안 시스템 운영

### 5. 영향 평가 결과

#### 5.1 개인정보 침해 위험도
- **높은 위험**: 계좌번호, 카드번호 등 금융정보
- **중간 위험**: 이름, 이메일 등 개인식별정보
- **낮은 위험**: 서비스 이용 통계 정보

#### 5.2 위험 완화 효과
- **기술적 조치**: 95% 위험 감소
- **관리적 조치**: 90% 위험 감소
- **물리적 조치**: 98% 위험 감소

#### 5.3 잔여 위험
- **잔여 위험 수준**: 낮음
- **추가 조치 필요성**: 없음 (현재 조치로 충분)

### 6. 개인정보 보호 영향 평가 의견
본 SaveAI 서비스의 개인정보 처리 활동은 개인정보보호법 및 관련 가이드라인을 준수하며, 적절한 보안 조치를 통해 개인정보 침해 위험을 최소화하고 있습니다. 추가적인 보안 조치가 필요하지 않으며, 서비스 운영이 가능하다고 판단됩니다.
